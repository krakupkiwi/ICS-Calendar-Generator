# Data Model

## Overview

The system uses a **flat row model** internally, which maps directly to/from CSV.

Each row represents **one grade on one match day**.

Multiple rows with the same `date` + `opponent` are combined into a single ICS calendar event.

---

## Fixture Row

```typescript
interface FixtureRow {
  id:       string;   // UUID, auto-generated
  date:     string;   // YYYY-MM-DD  (e.g. "2026-05-12")
  round:    string;   // Human label (e.g. "Round 5", "Finals Week 1")
  opponent: string;   // Opposing club (e.g. "Norths")
  location: string;   // "Home", "Away", or venue name
  grade:    string;   // Team/competition (e.g. "First Grade", "Womens League Tag")
  time:     string;   // HH:MM 24h (e.g. "15:30") — empty if TBC
}
```

---

## CSV Format (source of truth)

The CSV format is the canonical external representation:

```csv
Date,Round,Opponent,Location,Grade,Time
2026-05-12,Round 5,Norths,Home,Womens League Tag,11:00
2026-05-12,Round 5,Norths,Home,First Grade,15:30
```

Column names are **case-insensitive** during import.

## ICS Event (generated)

Multiple rows with the same `date` + `opponent` become one ICS event:

```
BEGIN:VEVENT
UID:fixture-2026-05-12-norths@rugbyleaguecal
DTSTAMP:20260101T000000Z
DTSTART:20260512T110000
DTEND:20260512T173000
SUMMARY:vs Norths [HOME] - Round 5
DESCRIPTION:Round 5\n\nKICK-OFF TIMES:\n  • Womens League Tag: 11:00am\n  • First Grade: 3:30pm\n\nLocation: Home\n\nGenerated by Rugby League ICS Calendar Generator
LOCATION:Home
END:VEVENT
```

### UID Generation

UIDs are stable and deterministic:
```
fixture-{date}-{opponent-slug}@rugbyleaguecal
```

Where `opponent-slug` lowercases the opponent name and replaces non-alphanumeric characters with hyphens.

Examples:
- `fixture-2026-05-12-norths@rugbyleaguecal`
- `fixture-2026-05-09-mt-gravatt@rugbyleaguecal`

**Why stable UIDs matter:** If you re-export and re-upload the ICS file, calendar apps (Apple Calendar, Google Calendar, Outlook) use the UID to identify events. Stable UIDs mean existing calendar subscriptions will receive updates rather than creating duplicate entries.

---

## Event Time Calculation

For a fixture with multiple grades:

| Grade | Kickoff |
|-------|---------|
| Womens League Tag | 11:00 |
| Reserve Grade | 13:00 |
| First Grade | 15:30 |

**ICS Event:**
- `DTSTART` = 11:00 (earliest kickoff)
- `DTEND` = 17:30 (latest kickoff 15:30 + 2 hour buffer)

This gives the full "day at the footy" window in calendar apps.

---

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Row with no time | Defaults to 10:00 for ICS generation; displayed as "TBC" in UI |
| Row with no grade | Included in fixture but skipped in description grade list |
| Duplicate rows (same date/opponent/grade) | Both included; displayed twice in description |
| Non-standard location | Passed through as-is to LOCATION field |
| Long opponent name with special chars | Slugified safely for UID |
